# Singleton 单例模式
* 定义
    - 一个类只允许创建一个对象(实例)，这个类就叫单例类，这种设计模式就叫单例设计模式，简称单例模式
* 用处
    - 在业务概念上，有些数据在系统中只应该保存一份，就比较适合设计为单例类
* 单例模式的唯一性
    - 线程间唯一
    - 进程间唯一
    - 集群环境中唯一
        - 通过外部共享存储的锁进行，如文件等
* 存在的问题
    - 对OOP的特性支持不友好
    - 会隐藏类之间的依赖关系
    - 对代码扩展性不友好
    - 对代码的可测试性不友好
    - 不支持由参数的构造函数
